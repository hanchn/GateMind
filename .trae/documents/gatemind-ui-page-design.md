## 1. 页面设计目标

本文件用于约束 GateMind 后台 UI 的页面结构与视觉规则，保证后续扩展页面时仍然保持统一、可维护和高辨识度。

设计目标：

- 先保证治理后台“信息清晰、状态可见、操作明确”
- 再强化企业 AI 风控中台的品牌感与控制台气质
- 保持组件复用和视觉规则稳定，避免页面越做越乱

## 2. 整体布局

后台采用桌面优先的三层结构：

1. 左侧固定导航
2. 顶部状态栏
3. 主内容工作区

```text
+-----------------------------------------------------------+
| 顶部状态栏：环境 / 搜索 / 用户 / 系统状态                  |
+-------------+---------------------------------------------+
| 左侧导航     | 主内容区                                     |
| 控制台       | 页面标题                                     |
| 工具目录     | 概览卡 / 筛选器 / 表格 / 图表 / 详情抽屉      |
| 策略规则     |                                             |
| 变更审批     |                                             |
| 审计日志     |                                             |
| 业务域       |                                             |
+-------------+---------------------------------------------+
```

## 3. 页面清单

### 3.1 控制台首页

页面目标：

- 让用户第一眼看到平台当前状态
- 突出风险、审批、异常和工具活跃度

页面结构：

- 品牌与欢迎区
- 指标卡片区
- 风险分布区
- 待审批区
- 热门工具区
- 最近异常区

### 3.2 工具目录页

页面目标：

- 让用户快速查看并管理工具元数据

页面结构：

- 页面标题与操作按钮
- 顶部筛选栏
- 工具列表表格
- 右侧工具详情抽屉

### 3.3 策略规则页

页面目标：

- 让治理规则配置清晰可控

页面结构：

- 页面标题
- 策略列表
- 条件标签区
- 规则详情 / 编辑面板

### 3.4 变更审批页

页面目标：

- 让审批人高效判断是否通过变更

页面结构：

- 风险摘要区
- 待审批表格
- 详情面板
- 审批操作区

### 3.5 审计日志页

页面目标：

- 让审计人员快速检索链路并查看快照

页面结构：

- 高级筛选栏
- 审计表格
- Trace 时间线详情

### 3.6 业务域页

页面目标：

- 让管理员看到各业务域接入和健康状态

页面结构：

- 业务域统计卡
- 域卡片网格
- Provider 状态列表

### 3.7 数据连接页

页面目标：

- 让管理员统一查看数据库连接的状态、环境、驱动和审批来源

页面结构：

- 连接状态概览
- 连接列表
- 连接详情面板

### 3.8 连接申请页

页面目标：

页面结构：

- 连接申请表单
- 申请说明卡
- 申请记录列表

### 3.9 库表治理页

页面目标：

- 让库表纳管、敏感字段治理和增删操作入口统一可见

页面结构：

- 库表统计区
- 库表清单
- 治理详情面板

### 3.10 个人 Token 页

页面目标：

- 让个人用户可以创建、查看、复制和吊销自己的访问 Token

页面结构：

- Token 安全说明区
- Token 创建面板
- Token 列表

## 4. 组件复用规则

以下组件必须沉淀为通用组件，不允许各页面重复造轮子：

- 页面头部 `PageHeader`
- 卡片容器 `SectionCard`
- 风险标签 `RiskBadge`
- 状态标签 `StatusBadge`
- 通用表格 `DataTable`
- 空状态 `EmptyState`
- 筛选工具栏 `FilterBar`
- 详情抽屉 `DetailDrawer`

## 5. 视觉规范

### 5.1 背景系统

- 页面大背景使用深色渐变与微弱网格纹理
- 主内容卡片使用半透明深底 + 细描边
- 高风险区域可增加更强对比底色

### 5.2 字体层级

- 页面标题：强识别度标题字体
- 区块标题：中等粗细
- 正文与表格：高可读正文无衬线字体
- 注释与辅助信息：较小字重，降低视觉噪音

### 5.3 色彩规则

- 中性色作为主要背景和内容承载
- 仅保留两类强调色：荧光青、信号橙
- 风险色独立编码，不与品牌强调色混用

## 6. 动效规则

- 页面进入时仅做轻量分层淡入
- 卡片 hover 使用微位移与阴影增强
- 抽屉采用右侧滑入
- 风险状态变化使用颜色过渡，不做夸张动画

原则：

- 动效必须增强信息感知，而不是炫技
- 同类组件使用相同动效节奏

## 7. 可维护性规则

- 新页面必须复用已有布局骨架与基础组件
- 页面结构优先通过组合实现，不复制粘贴已有模块
- 表格、标签、筛选器、抽屉统一收口到公共组件
- 视觉变量只能来自统一设计令牌
- 新增业务页面时先判断是否属于已有页面扩展，而不是直接新增新范式页面
- 安全相关页面如 Token、连接申请必须优先突出权限范围、过期时间和审批状态

## 8. 当前实现优先级

UI 第一阶段按以下顺序实现：

1. 后台框架布局
2. 控制台首页
3. 工具目录页
4. 变更审批页
5. 审计日志页
6. 策略规则页
7. 业务域页
8. 数据连接页
9. 连接申请页
10. 库表治理页
11. 个人 Token 页

这样可以先搭出最核心的中台骨架，再补齐治理细节页面。
